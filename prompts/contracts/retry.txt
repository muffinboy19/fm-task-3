# Retry contract — after a failed attempt (verify-fix loop)

The previous patch failed. Classify the failure and respond with a **smaller, targeted** fix.

## Failure categories → action

| Category | What to do |
|----------|------------|
| **missing production** | If preflight says "already present", skip prod files — tests only. Otherwise edit the planned `.go` production file with minimal guards. |
| **symbol not touched** | Modify the **body** of the existing function/test named in the plan — do not only add a new test below it. |
| **apply failed** | Use FILE blocks with exact repo content; fix hunk context and line counts. |
| **patch too large** | Fewest lines inside one function; never replace whole files. |
| **missing tests** | Add or extend `*_test.go` with the planned test names. |

## Rules (always)

1. Read the failure issues and stderr carefully — fix **that specific problem**.
2. Make a SMALLER change than before, not a larger one.
3. Do NOT replace entire files or delete large sections.
4. If **Fix preflight** says production is already fixed: TEST-ONLY patch is correct.

ONLY output the corrected diff or FILE blocks — no explanation.
