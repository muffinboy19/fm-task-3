# Output contract — fix plan

Focus only on what the issue describes. Identify the minimal files and functions to change.

Rules:
- Name concrete *_test.go tests to add or update.
- If the issue is ambiguous, note assumptions in one short bullet list.
- Do not plan refactors, renames, or unrelated cleanup.

Output format (markdown). Short bullets only — one line each. No tables, no long paragraphs.
